Render-Blocking & Download-Priorität im Network Tab

mobile.css

Kein media-Attribut → Highest, render-blocking. Das ist das Fundament, auf das der erste Paint wartet.

desktop.css

media="(min-width:1024px)". Auf dem Handy: Lowest, wird geladen, blockiert den Paint nicht.

print.css

media="print". Am Bildschirm: Lowest, blockiert nie. Der eindeutige Extremfall.

Was du im Network Tab beobachtest

  1. DevTools → Network → Spalte Priority einblenden (Rechtsklick auf einen Spaltenkopf → Priority anhaken).
  2. Device-Toolbar an (Cmd/Ctrl+Shift+M), Gerät „iPhone SE" wählen, neu laden.
  3. mobile.css = Highest, desktop.css & print.css = Lowest.
  4. Auf „Responsive / 1440px" umstellen, neu laden → desktop.css springt auf Highest (jetzt render-blocking).
