Responsive Images: das Handy lädt weniger Bytes

Der Browser wählt vor dem Download anhand von Viewport & sizes die passende Datei aus srcset. Ein Smartphone lädt photo-480.jpg (klein) statt photo-2000.jpg (groß) – obwohl der HTML-Code identisch ist.

Ungefähre Dateigrößen (echt, in diesem Ordner)

DateiAuflösungGrößegeladen bei
photo-480.png480×32036 KBschmales Handy
photo-1200.png1200×800312 KBTablet / kleiner Desktop
photo-2000.png2000×1333948 KBgroßer / Retina-Desktop

Was du im Network Tab beobachtest

  1. DevTools → Network, Filter Img, Cache leeren (Disable cache anhaken).
  2. Device-Toolbar an, „iPhone SE" → neu laden → es lädt photo-480.jpg (~36 KB).
  3. Auf „Responsive" 1440px stellen → neu laden → jetzt photo-2000.jpg (~948 KB).
  4. Vergleich: 36 KB vs. 948 KB = ~26× weniger Daten fürs Handy – allein durch die richtige Bildwahl. So viel spart keine Media-Query-Schreibrichtung je ein.

Tipp: Der Effekt greift nur beim ersten Laden pro Größe → immer „Disable cache" an lassen und hart neu laden.
